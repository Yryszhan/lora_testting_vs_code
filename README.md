# QUASAR Logic — Custom LoRaWAN Node

Промышленный LoRaWAN-узел на базе **STM32F103** и модуля **Ai-Thinker Ra-01SH (SX1262)**, разработанный с нуля для мониторинга оборудования нефтедобычи (нефтяные качалки) компании munaitelecom.

Узел реализует полный стек **без использования сторонних LoRa/LoRaWAN библиотек**: драйвер радиочипа на голом SPI, собственная реализация AES-128 с расчётом MIC и шифрованием payload, сборка корректного LoRaWAN ABP-кадра.

---

## Ключевые особенности

- **Голый драйвер SX1262** — управление радиочипом напрямую через SPI-команды, без RadioLib/LMIC
- **Собственный AES-128** — S-box, расширение ключа, шифрование блока (проверено на тест-векторах FIPS-197)
- **LoRaWAN-крипта с нуля** — AES-CMAC для MIC (RFC 4493) и AES-CTR для шифрования payload
- **ABP-активация** — DevAddr + NwkSKey + AppSKey, инкрементируемый FCnt
- **Полевые испытания** — подтверждённая дальность **до 700 м в городской застройке**
- Интеграция с **ChirpStack** через SX1302-gateway на Raspberry Pi

## Аппаратное подключение

### Радиомодуль Ra-01SH (чип SX1262)

| Пин Ra-01SH | Сигнал | Подключение к STM32 |
|:-----------:|:-------|:--------------------|
| 3  | 3.3V (VCC)  | Питание 3.3 В (через AMS1117) |
| 2, 9, 16 | GND | Земля |
| 15 | NSS  | GPIO (Chip Select) |
| 14 | MOSI | SPI1 MOSI |
| 13 | MISO | SPI1 MISO |
| 12 | SCK  | SPI1 SCK |
| 4  | RESET | GPIO |
| 6  | DIO1 | GPIO (вход, прерывание TxDone) |
| 10 | BUSY | *не разведён* (заменён задержками) |
| 5  | TXEN | **N/C** (свитч управляется через DIO2) |
| 11 | RXEN | **GND** (закрытие RX-плеча, CTRL = 0) |

### Параметры SPI

- Интерфейс: **SPI1**, Mode 0 (CPOL = 0, CPHA = 0)
- Делитель: prescaler 16
- NSS: программный (Software NSS)

### Управление RF-свитчем

Антенный свитч на модуле управляется **внутренне через DIO2 чипа** SX1262:

```
DIO2 (чипа) → R8 (0R) → транзистор Q1 → RF-свитч
DIO2 = 1, CTRL = 0 → Tx Mode
DIO2 = 0, CTRL = 1 → Rx Mode
```

Активируется командой `SetDIO2asRfSwitch (0x9D 0x01)`. Внешние выводы TXEN/RXEN на этом модуле **не требуются** (подтверждено схемой модуля и библиотекой nopnop2002).

### Радиопараметры

| Параметр | Значение |
|:---------|:---------|
| Частота | 868.1 МГц (EU868, канал 0) |
| Spreading Factor | SF12 |
| Bandwidth | 125 кГц |
| Coding Rate | 4/5 |
| Мощность TX | +22 dBm |
| Регулятор | LDO (не DC-DC) |
| Sync Word | 0x3444 (LoRaWAN public) |
| TCXO | отсутствует (только кварц) |

### Низкоуровневый драйвер SX1262

| Функция | Назначение |
|:--------|:-----------|
| `busy()` | Программная задержка вместо аппаратного BUSY |
| `sx_cmd(tx, rx, len)` | Передача SPI-команды (NSS↓ → обмен → NSS↑) |
| `sx_reset()` | Аппаратный сброс через RST |
| `sx_chipmode()` | Чтение режима чипа (`0xC0`) — диагностика |
| `sx_readreg(addr)` | Чтение регистра (`0x1D`) — проверка сигнатуры |
| `sx_radio_config()` | Полная конфигурация радио при старте |
| `sx_send_packet()` | Сборка кадра и передача |

### Криптографический стек (AES-128)

| Функция | Назначение |
|:--------|:-----------|
| `aes_key_expand()` | Расширение ключа (11 round-ключей) |
| `aes_encrypt_block()` | Шифрование блока 16 байт |
| `aes_cmac()` | AES-CMAC (для расчёта MIC) |
| `cmac_subkey()` | Генерация подключей K1/K2 |
| `lorawan_encrypt_payload()` | Шифрование payload (AES-CTR, AppSKey) |
| `lorawan_compute_mic()` | Расчёт MIC (CMAC по блоку B0, NwkSKey) |
| `lorawan_build_uplink()` | Сборка полного LoRaWAN-кадра |

### Структура LoRaWAN-кадра (uplink)

```
+------+---------+-------+------+-------+------------------+-----+
| MHDR | DevAddr | FCtrl | FCnt | FPort | FRMPayload (enc) | MIC |
| 0x40 | 4 байта | 0x00  | 2 б  | 0x01  |   зашифрован     | 4 б |
+------+---------+-------+------+-------+------------------+-----+
```

## Конфигурация ChirpStack (ABP)

Устройство активируется в режиме **ABP** (Activation By Personalization):

- **DevAddr** — адрес устройства (4 байта)
- **NwkSKey** — Network Session Key (16 байт) — для проверки MIC
- **AppSKey** — Application Session Key (16 байт) — для расшифровки payload

На время отладки рекомендуется включить **"Disable frame-counter validation"** — узел может перезагружаться (сброс FCnt), и без этой опции сервер отбрасывает кадры.

> **Для продакшна:** FCnt необходимо хранить в энергонезависимой памяти (Flash), чтобы счётчик переживал перезагрузки.

## История отладки (lessons learned)

Путь от «чип молчит» до рабочего зашифрованного узла прошёл через несколько неочевидных проблем:

1. **XOSC_START_ERR.** Код отправлял команду TCXO (`SetDIO3asTCXOCtrl`), но Ra-01SH **не имеет TCXO** — только кварц. Лишняя команда блокировала запуск генератора. Решение: убрать команду TCXO.

2. **RF-свитч.** Долгое выяснение, что свитч управляется внутренне через DIO2 (измерено мультиметром: DIO2 = 2.9 В при передаче), а внешняя перемычка TXEN→3.3В конфликтовала с этим управлением через R8. Решение: снять перемычку, RXEN на GND.

3. **Регулятор питания.** Стоял DC-DC (`0x96 0x01`), а Ra-01SH использует **только LDO**. Решение: `0x96 0x00`.

4. **Смещение частоты (главная причина CRC_FAIL).** Код задавал 868.135 МГц вместо 868.100 — смещение **+35 кГц**. При SF12 это приводило к потере пакетов. Решение: точная частота `0x86 0x36 0x40 0x3D 0x60` = 868.100 МГц. После этого пакеты пошли с CRC_OK и стали форвардиться в ChirpStack.

5. **LoRaWAN-крипта.** Изначально MIC = `0x00000000` (заглушка), payload в открытом виде — ChirpStack отбрасывал кадр на валидации MIC. Решение: реализован полный AES-стек, MIC сошёлся, "HI" появился в Device Data.

## Результаты полевых испытаний

| Расстояние (город) | Результат |
|:-------------------|:----------|
| до ~400 м | уверенный приём |
| 500–700 м | рабочая зона (доходимость 25–75%) |
| 700 м | RSSI до −115 dBm, SNR до −11 dB — на пределе чувствительности SF12 |

**Наблюдение:** при нагреве кварца частота уходила (foff с −5.8 до +20 кГц) — узел «гулял» по каналам, но gateway ловил его на соседних. Для стабильности на улице (перепады температуры) в следующей ревизии планируется **модуль с TCXO**.

**Подъём gateway-антенны на 12 м** резко улучшил приём — подтвердил критическую роль высоты в городской застройке.

## Технологический стек

- **MCU:** STM32F103 (HAL, генерация CubeMX)
- **Радио:** Ai-Thinker Ra-01SH (Semtech SX1262)
- **Gateway:** Raspberry Pi + SX1302 concentrator HAT
- **Network Server:** ChirpStack (EU868)
- **Шифрование:** собственная реализация AES-128 (CMAC + CTR)
- **CAD:** KiCad
- **Производство:** JLCPCB, ручная пайка с трафаретом
